Java Agent Container

Java Agent Container is an abstract layer that separates common tasks from applications, thus saves programmers from low level programming such as threading logic, low level JMS logic, socket logic, configuration logic, and etc.
It is xml file driven, all resources are defined by config.xml. Applications are plug-in classes that are defined in the config.xml. Resources are loaded at start up time, and are visible to applications. Requests are accepted by the Container and passed to the application processors. Application programmers need only to focus on the business logic.
It uses thread pooling to reduce response time since the worker thread is already created, started, and running. It is only waiting for the signal to go!
Four general types of applications can be developed and deployed under the Container: 1) Queue driven applications, 2) Socket driven applications, 3) pop3/imap driven applications, 4) Applications that have no driving source
It can be used together with other existing frameworks like spapi, mq wrapper, gapi, qapi, etc.
This page represents an test implementation of the Container. Two servers are running under the Container, one is Queue server that serves queue messages, another is a simple HTTP server that displays this page and Agent's runtime status.
